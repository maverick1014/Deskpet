# 宠物屏幕行走系统

> 返回 [文档索引](./README.md)

---

## 概述

宠物以悬浮窗形式显示在 Android 屏幕上，能够自主行走、停留、做出各种动作。行走行为受**性格**和**当前情绪**驱动，不同宠物的走法、频率和习惯都不一样。

---

## 悬浮窗实现

### 技术基础

| 项目 | 技术方案 |
|------|----------|
| 悬浮显示 | Android Overlay（`SYSTEM_ALERT_WINDOW` 权限） |
| 窗口管理 | WindowManager 添加/更新 View |
| 渲染 | Flutter Engine 嵌入悬浮窗 |
| 动画 | Sprite Sheet / Lottie / Rive 动画 |

### 权限要求

- `SYSTEM_ALERT_WINDOW` — 悬浮窗权限
- 首次启动引导用户授权

## 行走行为

### 行走模式

| 模式 | 说明 |
|------|------|
| 自由行走 | 宠物在屏幕边缘/底部随机走动 |
| 待机停留 | 在某个位置停下，播放待机动画 |
| 跟随手指 | 玩家拖拽宠物到指定位置 |
| 边缘依附 | 宠物沿屏幕边缘行走（像在走钢丝） |
| 躲藏 | 宠物躲到屏幕边缘只露出一部分 |

### 性格对行走的影响

| 性格 | 行走表现 |
|------|----------|
| 活泼度高 | 走动频繁、速度快、走路蹦蹦跳跳 |
| 活泼度低 | 走动少、速度慢、更多时间待机 |
| 胆量高 | 大范围行走、不怕屏幕中央 |
| 胆量低 | 倾向待在屏幕角落、不敢走太远 |
| 亲密度倾向高 | 常走向通知栏/状态栏方向（靠近玩家视线） |
| 好奇心高 | 喜欢探索屏幕不同区域 |

### 情绪对行走的影响

| 情绪 | 行走表现 |
|------|----------|
| 开心 | 蹦蹦跳跳、转圈、走得欢快 |
| 伤心 | 缓慢移动、低头走路、经常停下 |
| 生气 | 走路用力跺脚、背对玩家方向走 |
| 无聊 | 漫无目的游荡、时不时戳屏幕 |
| 害怕 | 快速跑到屏幕角落躲起来 |
| 疲惫 | 几乎不走动、原地打瞌睡 |

## 动画系统

### 动画状态

| 状态 | 动画描述 |
|------|----------|
| 待机 | 呼吸动画、眨眼、微小动作 |
| 向左走 | 面朝左的走路动画 |
| 向右走 | 面朝右的走路动画 |
| 跑步 | 快速移动动画 |
| 跳跃 | 蹦跳动画 |
| 坐下 | 坐下休息动画 |
| 睡觉 | 打瞌睡 / 躺下睡觉 |
| 开心 | 开心表情 + 特殊动作 |
| 伤心 | 伤心表情 + 蜷缩动作 |
| 生气 | 生气表情 + 跺脚 |
| 说话 | 嘴部动画（配合语音助手） |
| 被拖拽 | 被抓起来的动画 |
| 被放下 | 落地动画 |

### 动画技术选型

| 方案 | 说明 | 优缺点 |
|------|------|--------|
| Sprite Sheet | 逐帧动画图集 | 简单可控、文件体积大 |
| Lottie | After Effects 导出 JSON 动画 | 矢量平滑、Flutter 支持好 |
| Rive | 交互式动画引擎 | 支持状态机、体积小、交互性强 |
| Spine | 骨骼动画 | 适合角色动画、学习成本高 |

> **推荐**：Rive 或 Lottie 作为主要动画方案，Rive 的状态机特性非常适合根据情绪切换动画。

## 触摸交互

### 交互方式

| 操作 | 效果 |
|------|------|
| 单击 | 宠物转头看玩家、播放回应动画、可能冒出对话气泡 |
| 双击 | 进入快捷菜单（喂食/游戏/状态） |
| 长按 | 进入语音助手模式 |
| 拖拽 | 拖动宠物到屏幕其他位置 |
| 甩出 | 快速滑动松手 → 宠物飞出去再跑回来（趣味互动） |
| 摸头（向下滑） | 宠物开心、快乐度微增 |
| 戳肚子（快速点击） | 宠物生气或咯咯笑（取决于性格） |

### 触摸 + 情绪联动

| 情绪状态 | 被触摸时的反应 |
|----------|----------------|
| 开心 | 撒娇、蹭手、开心叫 |
| 生气 | 躲开、拒绝互动、哼一声 |
| 伤心 | 抬头看玩家、慢慢靠近 |
| 疲惫 | 不太想动、打哈欠 |

## 对话气泡

### 触发方式

宠物会在特定情况下自动冒出对话气泡：

| 触发条件 | 气泡内容示例 |
|----------|-------------|
| 待机一段时间 | "..." "好无聊" "zzZ" |
| 情绪变化 | "今天心情不错！" "我有点难过..." |
| 被触摸 | "嘿嘿~" "干嘛啦！" |
| 时间事件 | "早上好！" "该睡觉了吧？" |
| 主动搭话 | "主人～" "你在忙什么？" |

### 气泡样式

- 轻量级气泡，显示 2~3 秒后自动消失
- 气泡内容受性格和情绪影响（活泼的宠物用语更可爱、内向的宠物用语更简短）
- 点击气泡可进入对话模式

## 屏幕适配与性能

### 行走区域

- 默认行走区域：屏幕底部（模拟在桌面上走）
- 可选模式：全屏自由行走（爬墙、走顶部）
- 自动避让输入法、通知栏

### 性能优化

| 优化点 | 方案 |
|--------|------|
| 电量消耗 | 宠物不在前台时降低动画帧率 |
| 内存占用 | 按需加载动画资源，闲置时释放 |
| 刷新频率 | 行走中 30fps，待机时 15fps |
| 遮挡处理 | 宠物不遮挡通知、不影响正常操作 |
| 点击穿透 | 非宠物区域的触摸事件穿透到底层 App |
