# 语音助手系统

> 返回 [文档索引](./README.md)

---

## 概述

桌宠具备类似 Siri 的语音助手功能。玩家通过呼唤宠物名字唤醒宠物，宠物会用语音回应并询问玩家需要什么帮助。宠物的说话方式、语气和内容受其**性格**和**当前情绪**影响，每只宠物的语音助手体验都是独特的。

---

## 唤醒机制

### 唤醒方式

| 方式 | 说明 |
|------|------|
| 语音唤醒 | 玩家说出宠物名字（如："小白"） |
| 点击唤醒 | 点击桌宠悬浮窗 |
| 长按唤醒 | 长按桌宠进入助手模式 |

### 语音唤醒流程

```
玩家呼唤宠物名字
    │
    ▼
本地唤醒词检测（始终监听，低功耗）
    │
    ▼
匹配成功？──否──→ 继续监听
    │
   是
    │
    ▼
宠物唤醒动画 + 语音回应
    │
    ▼
进入对话模式（开始录音/识别）
    │
    ▼
玩家说出指令 / 闲聊
    │
    ▼
AI 理解 + 执行 / 回复
    │
    ▼
超时无输入 → 宠物告别语 → 回到待机
```

### 唤醒词规则

- 默认唤醒词 = 玩家给宠物取的名字
- 支持自定义唤醒词（备选名/昵称）
- 唤醒词长度建议 2~4 个字（提高识别准确率）

## 语音交互能力

### 宠物能做的事

| 类别 | 指令示例 | 执行动作 |
|------|----------|----------|
| **状态查询** | "你还好吗？"、"你饿了吗？" | 报告当前健康/饱腹/清洁/快乐数值 |
| **喂食** | "吃饭了"、"给你吃东西" | 打开食物选择界面 / 直接喂食 |
| **吃药** | "吃药"、"去看医生" | 打开药物界面（仅生病时有效） |
| **工作** | "去工作"、"上班了" | 开始工作任务 |
| **学习** | "去上课"、"去学校" | 开始上课 |
| **游戏** | "来玩游戏"、"我们来玩" | 打开小游戏选择 |
| **洗澡** | "去洗澡"、"洗一下" | 执行清洁操作 |
| **闲聊** | "今天怎么样？"、"你开心吗？" | 根据当前情绪和性格进行对话 |
| **天气** | "今天天气怎么样？" | 查询并播报天气 |
| **提醒** | "提醒我下午3点开会" | 设置闹钟/提醒 |
| **时间** | "现在几点？" | 报时 |

### 拒绝与个性化回应

宠物不是无条件听从的工具，会根据**情绪和性格**做出不同反应：

| 情境 | 高信任 / 好情绪 | 低信任 / 坏情绪 |
|------|------------------|------------------|
| 叫去工作 | "好的，我去啦~" | "我不想去...好累..." |
| 叫去学习 | "嗯！我要好好学习！" | "又要上课...能不能休息一下" |
| 闲聊 | 热情回应，语气活泼 | 冷淡回应，语气低落 |
| 生病时被叫去工作 | "我有点不舒服，但我试试..." | "我生病了...你不关心我吗？" |

> 宠物最终仍会执行指令（除非生病静养期），但语气和态度体现个性。

## 语音合成（TTS）

### 性格对语音的影响

| 性格特征 | 语音表现 |
|----------|----------|
| 活泼度高 | 语速较快、语调上扬、用词活泼 |
| 活泼度低 | 语速较慢、语调平缓、用词简洁 |
| 胆量高 | 声音自信、干脆 |
| 胆量低 | 声音柔弱、带犹豫 |
| 亲密度倾向高 | 爱用语气词（"嘛~"、"呀~"） |
| 亲密度倾向低 | 简短回应、较少撒娇 |

### 情绪对语音的影响

| 当前情绪 | 语音表现 |
|----------|----------|
| 开心 | 声音明亮、语速稍快 |
| 伤心 | 声音低沉、语速慢、可能带哽咽 |
| 生气 | 声音生硬、语气不耐烦 |
| 疲惫 | 声音无力、带打哈欠 |
| 害怕 | 声音发抖、语速不稳 |

## 对话管理

### 对话模式

| 模式 | 说明 |
|------|------|
| 指令模式 | 识别具体指令 → 执行对应游戏操作 |
| 闲聊模式 | 无明确指令时进入自由对话 |
| 主动搭话 | 宠物根据情绪主动发起对话（如饿了主动说"我好饿"） |

### 上下文管理

- 支持多轮对话（如："你饿了吗？" → "那给你吃什么好？"）
- 对话超时（10 秒无输入）→ 宠物说告别语 → 回到待机
- 对话历史影响短期记忆

### 主动搭话触发

| 触发条件 | 宠物可能说的话 |
|----------|----------------|
| 饱腹度 < 30 | "主人...我好饿..." |
| 快乐度 < 20 | "好无聊啊...能陪我玩吗？" |
| 健康值 < 40 | "我不太舒服..." |
| 长时间未互动 | "主人？你还在吗？" |
| 完成工作后 | "我工作完了！好累～" |
| 刚痊愈 | "我好多了！谢谢主人照顾我" |

## 技术实现

### 语音识别（STT）

| 方案 | 说明 | 优缺点 |
|------|------|--------|
| Android SpeechRecognizer | 系统内置语音识别 | 免费、离线支持有限 |
| Google Cloud Speech-to-Text | 云端识别 | 准确率高、需网络、有费用 |
| Vosk（离线） | 开源离线语音识别 | 免费、完全离线、中文支持 |
| Whisper（本地/云端） | OpenAI 模型 | 准确率极高、本地需算力 |

### 语音合成（TTS）

| 方案 | 说明 | 优缺点 |
|------|------|--------|
| Android TTS | 系统内置 | 免费、声音机械 |
| Google Cloud TTS | 云端合成 | 声音自然、需网络、有费用 |
| Edge TTS | 微软 Edge 的 TTS | 免费、声音自然、需网络 |
| 本地 TTS 模型 | 如 Piper/VITS | 离线、可自定义声音 |

### 唤醒词检测

| 方案 | 说明 | 优缺点 |
|------|------|--------|
| Porcupine (Picovoice) | 自定义唤醒词引擎 | 低功耗、支持自定义、有免费额度 |
| Snowboy | 开源唤醒词 | 免费、已停止维护 |
| 自训练模型 | 基于 TFLite | 完全自定义、开发成本高 |

### 对话理解（NLU）

| 方案 | 说明 | 适用阶段 |
|------|------|----------|
| 关键词匹配 + 意图规则 | 本地规则引擎 | Beta / MVP |
| Dialogflow / Rasa | 意图识别框架 | 中期 |
| LLM API（GPT / Gemini） | 大语言模型 | 高级对话 |

> **Beta 1 建议**：唤醒词用 Porcupine，语音识别用 Vosk（离线），TTS 用 Edge TTS，对话理解先用关键词匹配 + 规则。预留 LLM 接口供后续升级。

---

## 隐私与性能

- 唤醒词检测在本地运行，不上传音频
- 语音识别优先离线方案，降低隐私风险
- 后台常驻监听需优化电量消耗
- 提供关闭语音唤醒的选项（仅保留点击唤醒）
