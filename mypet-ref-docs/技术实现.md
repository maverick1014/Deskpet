## 开发环境

| 项目 | 技术选型 |
|------|----------|
| 平台 | Android |
| 前端 | Flutter |
| 后端逻辑 | Dart |
| 数据存储 | 本地 SQLite / SharedPreferences |
| 桌面悬浮显示 | Android Overlay / WindowManager |

---

## Stage 1 技术栈

### 悬浮窗与屏幕行走

| 项目 | 技术方案 |
|------|----------|
| 悬浮窗 | Android Overlay（`SYSTEM_ALERT_WINDOW`） |
| 窗口管理 | WindowManager |
| 渲染 | Flutter Engine 嵌入悬浮窗 |
| 动画 | Rive / Lottie |
| 触摸交互 | GestureDetector + 坐标计算 |

### AI 情感引擎

| 项目 | 技术方案 |
|------|----------|
| 情绪计算 | 规则引擎（状态机 + 权重表），预留 ML 接口 |
| 性格存储 | SQLite |
| 记忆系统 | SQLite（短期事件队列 + 长期统计） |

### 语音助手

| 项目 | 技术方案 |
|------|----------|
| 唤醒词检测 | Porcupine（Picovoice） |
| 语音识别（STT） | Vosk（离线） |
| 语音合成（TTS） | Edge TTS |
| 对话理解（NLU） | 关键词匹配 + 规则引擎，预留 LLM 接口 |

---

## Stage 2 技术栈

### 状态机设计

管理以下状态：

- 健康值、饱腹度、清洁度、快乐度
- 疾病阶段
- 学习进度
- 工作状态
- 死亡状态

### 时间与计时

- 基于真实时间
- 每次操作后检查状态变化

### 事件触发逻辑

| 事件 | 触发结果 |
|------|----------|
| 工作完成 | 健康检查 |
| 学习完成 | 解锁职业 |
| 小游戏完成 | 数值提升 |
| 静养结束 | 恢复状态 |
